Redundancy

Redundancy helps to eliminate single points of failure.

Design consideration

  • Secure file and mail servers in a centralized location.
  • Protect the location by physical and logical security measures.
  • Create redundancy in the server farm

Common Applications

Network-Aware Applications

  • Software programs that are used to communicate over the network.
  • software programs that have been designed to intelligently adapt their behavior based on the current network conditions

Application Layer Services

  • Programs that interface with the network and prepare the data for transfer.
  • It facilitates communication between different software applications

Protocols

Network Protocols Define:

  • Processes on either end of a communication session.
  • Syntax of the messages
  • How messages are sent and the expected response.

Common protocols

  • IP
  • TCP
  • UDP
  • HTTP - 80

  • SMTP - 25
  • FTP - 21

  • DHCP - 67 and 68

Real-Time Applications

Malware

Virus

Malicious software that is attached to another program to execute a particular unwanted function on a workstation

Trojan horse

n entire application written to look like something else, when in fact it is an attack tool

Worms

Worms are self-contained programs that attack a system and try to exploit a specific vulnerability in the target.

The worm copies its program from the attacking host to the newly exploited system to begin the cycle again.

Network Attacks

Categories of network attacks

  • Reconnaissance Attacks
  • Access Attacks
  • Social Engineering - Attempts to manipulate individuals into performing actions or divulging confidential information.
  • Denial of Service Attacks (DoS)

Mitigating Network Attacks

1.Backup, Upgrade, Update, and Patch

2.Authentication, Authorization, and Accounting (AAA)

  • authentication - Users and administrators must prove their identity
  • Authorization - Determines which resources the user can access and what operations
  • Accounting - Records what the user accessed, the amount of time the resource is accessed, and any changes made

3.Firewalls

  • resides between two or more networks
  • controls traffic and helps prevent unauthorized access.
  • methods used:
    • Packet Filtering
    • Application Filtering
    • URL Filtering
    • Stateful Packet Inspection (SPI)
